# 《桃源茶肆》Demo V0.4 UI 清理说明

## 版本目标

V0.4 主要解决当前界面“代码文字压在美术背景上”的问题，让 Demo 更接近可试玩界面。此次没有大改核心玩法，重点是整理顶部状态、操作台、按钮和座位显示。

## 已调整内容

1. 移除了代码生成的重复大标题，避免和背景图里的“桃源茶肆”招牌重叠。
2. 新增顶部半透明状态栏，等级、金币、成功/流失和提示信息都放到状态栏内，提高可读性。
3. 把提示文字移动到一个较小的半透明提示条中，减少对背景图的遮挡。
4. 缩小操作台面板，并移动到底部按钮区上方。
5. 缩小制作进度条，降低 UI 遮挡。
6. 将茶饮、升级、调试按钮固定到底部区域，并缩小按钮尺寸。
7. 隐藏自动生成座位的“座位1/座位2/座位3”文字，避免遮住桌椅图片。
8. 缩小默认座位图片尺寸，减少对背景中柜台和桌椅的覆盖。

## 当前界面结构

代码生成内容仍位于 `RuntimeRoot` 下。你手动放在 Canvas 下、但不在 `RuntimeRoot` 里的图片和锚点不会被清空。

推荐继续使用这些手动锚点：`EntrancePoint`、`ExitPoint`、`Seat_1`、`Seat_2`、`Seat_3`。如果它们存在，客人会使用你手动摆放的位置。

## 下一步建议

V0.5 建议把操作台和底部按钮也改成场景可视化节点。这样你可以在 Cocos 里直接拖动按钮和面板位置，而不是每次通过代码调坐标。
