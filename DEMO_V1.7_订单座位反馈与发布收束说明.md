# 《桃源茶肆》Demo V1.4-V1.7 迭代说明

## 版本目标

本轮从 V1.4 直接推进到 V1.7，目标不是扩张新系统，而是让已有可试玩 Demo 更稳定、更清楚、更适合对外试玩。处理优先级按“订单可读性、座位理解、试玩反馈、Debug 收束”推进。

## V1.4 订单气泡表现

订单气泡扩大到更易读的尺寸，并继续使用 `对话框.png` 作为气泡底图。气泡顶部新增状态色条，普通订单使用绿色提示，急单使用红色提示。客人还在走向座位时会显示“前往几桌”和即将点单的信息，入座后显示桌号、剩余秒数、茶饮、剩余杯数、进度和客人类型。急单时保留红色文字与“急”标识，避免贴图恢复后降低可读性。

## V1.5 桌椅与座位表现

每张桌椅新增状态标识。空座显示“空座”，有客人在等待时显示“等茶”，并通过桌椅颜色轻微区分占用状态。客人入座、离开、清档、打烊清场后都会刷新座位状态，方便玩家理解哪些桌正在等待、哪些桌可接待。

## V1.6 试玩反馈强化

升级失败时会明确显示还差多少金币，并提示继续接单或完成日目标。升级成功时会显示当前等级、座位/接待变化，并在解锁新茶饮时直接列出新茶饮名称。服务结算提示修复了乱码文案，收入显示改为包含基础收入、小费和事件奖励后的总收入，并在金币已够升级时提示玩家可以升级。队列满时茶饮按钮显示当前队列数量，反馈更具体。

## V1.7 Debug 与发布收束

调试按钮默认隐藏，保留一个 `Debug 显示/隐藏` 开关。对外试玩时界面不再默认露出刷客、清档、升一级按钮；开发测试时仍可一键显示调试按钮快速复现场景。这样不牺牲测试效率，也减少试玩者误点调试按钮导致流程被破坏的风险。

## 当前测试重点

运行 `assets/scene/Demo.scene` 后建议先清档测试一次自然流程：等待客人入座，观察气泡从“前往桌位”变成正式订单；完成订单后看桌椅是否从“等茶”恢复为“空座”；赚够金币后升级，确认升级提示是否展示新等级和新解锁内容；点击 Debug 开关确认调试按钮能显示和隐藏。

还需要重点回归：急单红色提示是否明显；多人订单进度是否正确；打烊结算后是否能点击新一天继续；Debug 默认隐藏后是否仍能通过开关清档；图片资源缺失时核心按钮和文字是否仍然可用。

## 下一轮建议

下一轮建议做 V1.8 数值与首局节奏二次校准。重点不是继续加新系统，而是用新存档连续试玩 3 天，记录第一天完成单数、收入、是否能升级、缺料次数、流失次数和日结评级。如果首日仍不够清楚，就继续调整目标、刷新速度、升级成本或提示；如果核心流程已经顺畅，再进入微信小游戏构建前检查。
