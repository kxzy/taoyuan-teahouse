# 《桃源茶肆》Demo V0.5 控制层说明

## 版本目标

V0.5 的目标是把按钮从运行画面层中独立出来，避免背景、操作台、进度条或运行时清理逻辑影响按钮显示和点击。现在按钮会生成到独立的 `ControlLayer`，而背景、座位、客人、操作台仍生成在 `RuntimeRoot`。

## 关键调整

- 新增 `ControlLayer`，专门承载绿茶、红茶、茉莉、升级、+300、刷客、清档、升1级按钮。
- `ControlLayer` 会被强制置顶，避免被背景和其他 UI 遮挡。
- `RuntimeRoot` 只负责背景、座位、客人、操作台和提示。
- 保留 `Seat_1`、`Seat_2`、`Seat_3`、`EntrancePoint`、`ExitPoint` 手动锚点支持。如果你在 Canvas 下创建这些节点，运行时会优先使用它们的位置。
- 优化了制作按钮反馈，点击茶饮后会显示当前队列数量。
- 优化了 +300 调试反馈，会显示当前金币。

## 测试重点

运行 `Demo.scene` 后，先确认右侧/中部按钮可以看到并点击。然后依次测试：刷客、绿茶制作、自动交付、金币增长、升级、红茶/茉莉解锁。

## 下一步建议

V0.6 可以开始恢复背景图，但要分层恢复：先只恢复背景，不恢复按钮贴图；确认按钮仍可点击后，再恢复桌椅、客人和对话框图片。这样每次只引入一个变量，避免再次出现按钮消失的问题。
