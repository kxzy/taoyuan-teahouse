# 研发推进记录

## 2026-06-28

本轮优先推进 V1.0 第一阶段的经营闭环，重点从“一键补货”改为可决策的正式补货商店，并补上缺料时的明确引导。

已完成：
- 新增补货商店面板：展示当前金币、今日采购次数、茶叶/糖/花/果四类补货项。
- 补货项支持独立价格、购买数量、当前库存和购买结果提示。
- 购买成功会扣金币、增加对应库存、刷新 HUD 与商店面板，并走延迟存档。
- 金币不足时会明确提示缺多少钱，并停留在补货商店。
- 制作茶饮缺料时，会明确指出缺少的资源，并自动打开补货商店，高亮第一项缺料。
- 新增茶谱图鉴基础页签：展示已解锁数量/总数、基础茶饮解锁状态、研发茶饮基础属性与销售统计。
- 研发成功后会刷新图鉴状态，停留在图鉴页时可立即看到最近新茶。
- 研发面板会直接显示图鉴已解锁/总数和最近新茶，强化研发与收集的正反馈。
- 补上 Combo / 完美火候最小骨架：连续上茶已有 Combo 金币奖励，制茶完成时会判定“完美火候”，上桌时给予轻量收入加成，并在 HUD / 成品台文本中显示反馈。
- 修正 `staffPanel` 类型声明缺失，避免员工页签相关代码在 TypeScript 编译时因属性不存在而失败。
- 修正图鉴统计为全量茶谱统计，列表仍保持轻量展示，超过展示数量时给出“还有更多”提示。
- 调整研发成功后的保存顺序，先结算研发相关成就奖励再落盘，避免奖励未及时保存。
- 图鉴页增加输入拦截，避免打开图鉴时误点到底层经营区域。
- 为运行时创建的 `UIManagerRoot` 补充固定 UITransform 尺寸，并统一弹窗/补货遮罩尺寸来源，降低动态 UI 对齐风险。
- 研发缺料时也会复用补货商店引导，避免研发系统停留在单句缺料提示。
- 新增 `V1.0_玩法闭环验收清单.md`，用于 Cocos Creator 预览时逐项验证补货、缺料、图鉴、研发联动和 Combo / 完美火候。
- 新增 `tools/validate-v1-loop.js` 与 `npm run validate:v1-loop`，提供无依赖静态验收，当前 19 项关键路径检查通过。
- 找到 Cocos Creator 3.8.8 自带 TypeScript 工具，新增 `tools/tsconfig.codex-check.json` 与 `npm run typecheck:cocos`；当前 Cocos 类型检查通过。
- 修复类型检查发现的 `Game.EVENT_HIDE`、Canvas 兼容字段、弹窗回调类型、顾客节点回收和存档合并空值推断问题。
- 扩展 `npm run validate:v1-loop` 到 19 项检查，新增 UIManager 架构边界、补货业务层处理、研发页图鉴进度和 typecheck 配置路径校验。
