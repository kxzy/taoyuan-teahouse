# 《桃源茶肆》Demo V0.6 背景恢复说明

## 版本目标

V0.6 按计划只恢复茶铺背景图，不恢复按钮贴图，也不恢复桌椅/客人/对话框图片，以便单独验证背景不会再次影响按钮显示和点击。

## 分层结构

当前 Canvas 下运行时会生成三层：

- `BackgroundLayer`：只放茶铺背景图。
- `RuntimeRoot`：放顶部状态栏、提示条、座位调试块、客人、操作台。
- `ControlLayer`：只放按钮，并强制置顶。

这个分层是为了避免背景图或其他 UI 遮挡按钮。后续恢复美术时，应按顺序恢复：背景 → 座位 → 客人 → 对话框 → 按钮贴图。每恢复一类素材，都先测试按钮是否仍然可见可点击。

## 本次测试重点

运行 `Demo.scene` 后，确认：

1. 茶铺背景图显示。
2. 底部两排按钮显示。
3. 按钮可以点击。
4. 点击“刷客”后客人生成。
5. 点击对应茶饮后制作队列变化。

如果按钮再次消失，说明背景层仍然影响坐标或层级，应立即回退背景，而不是继续恢复其他图片。
