# 《桃源茶肆》V2.5 UI 信息层级整理说明

## 版本目标

本轮根据产品设计和小游戏运营视角，优先解决界面信息层级混乱的问题。目标不是更换美术，而是把界面从“开发版原型”整理成更接近可测试小游戏的结构：顶部看状态，中部看顾客和订单，下方看制作与操作。

## 本次调整

1. 顶部状态栏精简：顶部改为天数、金币、口碑和当前提示，弱化店铺等级、连续数等开发型信息。
2. 顾客区重排：桌位从三列横排改为两列布局，更适合 6 桌版本范围，也方便后续扩展。
3. 订单卡片缩小：订单气泡宽度明显缩小，信息改为桌号/急单、茶饮、剩余秒数和进度，减少视线跳动。
4. 急单独立提示：新增左上急单区域，显示最紧急订单，避免急单只靠普通气泡变红而不够醒目。
5. 制作区独立强化：制作台面板加宽加高，位置固定在顾客区下方，进度条加长，制作台文案拆成更清楚的两行。
6. 操作区重排：茶饮按钮重排为上下两排，功能按钮集中到底部同一行，Debug 继续默认隐藏。
7. 原料信息收束：原料不再挤在主要引导条里，而是放在制作区和按钮区之间作为轻量状态信息。

## 当前测试重点

运行 `assets/scene/Demo.scene` 后，重点看：顶部状态是否更清楚；桌位是否按两列排列；订单气泡是否不会遮挡；急单区域是否能正确显示最急订单；制作台面板是否能完整显示两行文字；底部茶饮和功能按钮是否在目标屏幕内。

## 下一轮建议

下一轮建议继续做 V2.6：把茶饮按钮进一步视觉分组为“茶底”和“配料/高阶茶”，并根据解锁等级隐藏或弱化未解锁按钮。之后再进入微信小游戏构建前检查。
