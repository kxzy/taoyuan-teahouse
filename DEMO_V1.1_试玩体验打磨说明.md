# 《桃源茶肆》Demo V1.1 试玩体验打磨说明

## 版本目标

V1.1 的目标是把 V1.0 从“功能完整”继续推进到“更适合试玩”。这次不新增大型玩法系统，而是根据 V1.0 固定测试清单做静态回归检查，并优先处理底部按钮区域的试玩干扰问题。

## 静态回归结论

本轮检查确认当前主入口仍是 `assets/scene/Demo.scene` 与 `assets/scripts/AutoDemoGame.ts`。代码中保留了背景层、运行层、控制层、顶部状态、经营提示、操作台、成品台、结算面板、六种茶饮按钮、升级、补货、研发和新一天按钮。`GameConfig.ts` 中 6 种基础茶饮、20 级等级配置、4 类客人仍与当前按钮和解锁逻辑对应。

从代码级检查看，V1.0 测试清单要求的关键入口都存在：客人刷新、订单气泡、制作队列、自动上茶、成品台保鲜、流失口碑、补货、升级、研发、日结算、新一天、清档和资源路径加载。实际画面表现仍需要在 Cocos 预览中跑一遍确认，尤其是底部按钮是否超出屏幕、安全区是否遮挡、结算面板是否遮住正在查看的信息。

## 本次调整

1. 将三个调试按钮从正式经营按钮行中进一步收纳到底部右侧 Debug 区。
2. 调试按钮缩小为 92×34，并改成两行短文本：`Debug/刷客`、`Debug/清档`、`Debug/升1级`。
3. 新增一个淡色 `Debug` 标识，让玩家知道这些不是正式经营操作。
4. 正式经营按钮区现在更集中：茶饮、升级、补货、研发、新一天仍保持主要按钮尺寸；调试按钮保留但视觉权重下降。

## 当前建议测试重点

运行 `Demo.scene` 后，先看底部正式按钮是否仍能完整显示并点击，再看右下 Debug 区是否会被屏幕边缘裁切。如果 Debug 按钮在目标设备上超出画面，可以下一轮把它们改成隐藏式开关，或者只在开发构建中显示。

## 下一步建议

V1.2 建议做首局节奏优化。重点观察玩家能否在 30 秒内完成第一单、1 到 2 分钟内看到升级或日结。如果节奏偏慢，可以微调首日刷客速度、首日目标、初始金币或前两级升级成本。如果节奏合适，再进入 V1.3 美术和表现恢复。
