# V1.0 玩法闭环验收清单

## 静态检查

- 执行 `npm run validate:v1-loop`，应看到 `V1.0 loop validation passed: 19 checks`。
- 执行 `npm run typecheck:cocos`，应无 TypeScript 报错。
- 重新执行 web-mobile 构建后，`temp/codex-build*/web-mobile-codex/index.html` 应产出，并可在本地静态服务中打开主场景。
- 以上检查只验证关键代码路径和 Cocos 类型编译，不能替代 Cocos Creator 预览。
- 如需命令行构建，先关闭占用项目日志的 Creator 实例，再构建到 `temp/codex-build`；生成目录和 build log 均出现后才算构建通过。

## 补货商店

- 点击底部“补货”按钮，应打开独立补货商店，而不是直接增加库存。
- 面板应显示当前金币、今日采购次数，以及茶叶、糖、花、果四类补货项。
- 每个补货项应显示名称、当前库存、购买数量和价格。
- 金币足够时购买任一补货项，应扣除金币、增加对应库存，并刷新 HUD 与商店数字。
- 金币不足时购买，应提示还差多少金币，库存和金币不应变化。
- 点击“返回经营”后商店关闭，底层经营流程可继续操作。

## 库存不足引导

- 将某种原料消耗到不足后点击对应茶饮，应提示具体缺少的资源和数量。
- 缺料后应自动打开补货商店，并在第一项缺料上显示“急需”标记。
- 研发缺料时也应打开补货商店，引导补齐关键原料。

## 茶谱图鉴

- 点击底部“图鉴”页签，应显示茶谱图鉴面板。
- 图鉴顶部应显示已解锁数量 / 总数。
- 基础茶饮应显示已解锁信息或未解锁占位目标。
- 已研发茶饮应显示品阶、名称、售价和基础属性。
- 研发成功后切到图鉴，或停留在图鉴页研发成功，应看到最近新茶更新。
- 研发页摘要应同步显示图鉴已解锁/总数和最近新茶。

## Combo / 完美火候

- 连续上茶时 HUD 应显示 Combo 反馈，并在收入消息中显示 Combo 奖励。
- 制茶完成满足条件时，应出现“完美火候”反馈。
- 完美火候茶上桌时，应在收入消息中显示火候奖励。
- 客人流失或成品茶浪费后，Combo / 火候连段应中断。

## 存档与稳定性

- 补货购买、制作消耗、研发成功后退出重进，金币、库存和研发茶谱应保持一致。
- 补货商店、图鉴、研发结果弹窗打开时不应卡住主流程；关闭或切页后应能继续经营。
