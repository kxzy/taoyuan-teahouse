# 《桃源茶肆》Demo V1.3 茶饮按钮表现恢复说明

## 版本目标

V1.3 开始进入美术与表现恢复阶段，但仍然遵循“小步恢复、每次只引入一个变量”的原则。本次只增强茶饮按钮表现，不改客人、桌椅、对话框和背景层级，避免再次影响按钮可见性和点击稳定性。

## 本次调整

1. 为六种基础茶饮按钮增加左侧小徽章，让茶饮按钮不再只有文字。
2. 绿茶、红茶、茉莉按钮优先复用已有茶饮图片资源，其中绿茶和茉莉共用现有绿茶素材，红茶使用现有红茶素材。
3. 桂花、雪梨、桃酿暂时使用彩色文字徽章，避免在缺少正式图标时引入新资源依赖。
4. 茶饮按钮文字向右留出一个全角空格，避免和左侧徽章重叠。
5. 按钮仍然保留原本的纯色 Button 结构和点击事件，不改变 ControlLayer 层级，降低按钮失效风险。

## 当前测试重点

运行 `Demo.scene` 后，重点确认六个茶饮按钮都能看到左侧徽章，按钮文字没有被徽章遮住，点击区域仍然可用。尤其要测试绿茶、红茶、茉莉三个带图片资源的按钮，如果图片未加载，应检查 `assets/resources/image` 下的茶饮图片资源是否存在；即使图片缺失，核心按钮文字和点击逻辑也不应受影响。

## 下一步建议

V1.4 可以继续小步恢复“桌椅与对话框表现”。建议先恢复或优化对话框，因为它直接影响订单可读性；确认订单气泡稳定后，再恢复桌椅表现。每恢复一类素材，都应按 V1.0 测试清单至少测试：刷客、点单、制作、自动上茶、急单、日结算。
